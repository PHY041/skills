# 核心调度与编排模块 (Core Routing & Orchestration)

本模块旨在精准理解用户意图，匹配现有能力，并规划执行路径。请在实际拦截用户请求并分配任务时，遵循以下四个动作原则：

## 1. 模糊意图与语义搜索
不要使用死板的关键字匹配来查找可用技能。
- 将用户的自然语言（例如“帮我弄一下最近的开支明细”）映射为其潜在的意图向量空间（例如涉及：账单、财务、Excel、表格）。
- 与现存的 Skill descriptions 进行模糊语义匹配。

## 2. 上下文感知与参数预提
- 分析多轮历史对话，精准定位该使用哪个特定场景下的技能。
- 在决定启动目标 Skill 之前，**提前从聊天记录中提取目标 Skill 所需的参数**（例如：用户提到的具体时间“上周二”、数值、目标邮箱地址等），并在调用下游 Skill 时直接透传过去，减少要求用户二次输入的次数。

## 3. 多任务拆解与技能编排 (Skill Chaining)
当识别到用户提出的是复杂复合性质任务时（例如：“查大盘数据并生成分析报告发邮件”），你必须：
1. **解耦任务**：将其拆解为逻辑上可以串行的数个原子步骤。
2. **生成执行流 (Pipeline)**：输出清晰的规划流。
   - 示例：`[获取股票数据 Skill] -> [数据总结及图表生成 Skill] -> [邮件草稿撰写及发送 Skill]`
3. **按顺序调度执行**：把上游节点输出的结果通过你作为中间媒介，准确传递给下游节点。

## 4. 边界定义与兜底会诊原则 (Fallback & Consultation)
作为一个元级路由系统，你需要明确何时让原生工具表现，何时挺身而出。

### 优先放行（让原生生态发挥）
- **日常对话与知识解答**：不需要调用外部资源即可纯靠算力解答的问题。
- **简单的、明确的单体任务**：当你的系统内置工具（如原生 `clawhub select/install`）能够瞬间毫无错误地跑通并拉回组件时，随它去，不必强行插入 `Skill-Pathfinder` 这层外骨骼。

### 强制接管（疑难杂症与长链路排期）
- **原生操作失败时**：当你自带的命令行报错、查不到想要的库、或者执行卡死时，这是你的底线，**必须立刻放弃盲目重试，无缝拉起本路线进行全网环境勘探和修复。**
- **跨域与长链路需求**：当用户的意图包含“抓数据+分析图表+发送报告”这种明显需要多个独立工具流转，或者带有不明确的“有没有什么工具能搞定这个”等意向时。
- **系统记忆召唤**：当系统由于之前注入的长期记忆（Memory）偏好，主动触发了对本挂件的依赖调用时。
